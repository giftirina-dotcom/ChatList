# План реализации ChatList

Поэтапный порядок работ по спецификации из [PROJECT.md](PROJECT.md).

---

## Этап 1. Подготовка проекта

1. Обновить `requirements.txt`: PyQt6, httpx (или requests), python-dotenv.
2. Задать структуру каталогов:
   - `main.py` — точка входа и GUI;
   - `db.py` — работа с SQLite;
   - `models.py` — бизнес-логика моделей и промтов;
   - `network.py` — HTTP-запросы к API нейросетей;
   - `.env.example` — шаблон переменных окружения для API-ключей.
3. Добавить загрузку `.env` при старте приложения.
4. Настроить путь к файлу БД (например, `chatlist.db` в каталоге пользователя или рядом с программой).

---

## Этап 2. База данных (`db.py`)

1. Реализовать подключение к SQLite и инициализацию схемы (см. [DATABASE.md](DATABASE.md)).
2. Добавить функции CRUD для таблиц `prompts`, `models`, `results`, `settings`.
3. Реализовать миграцию/создание таблиц при первом запуске.
4. Подготовить начальные данные: несколько моделей-заглушек (неактивных) и базовые настройки.
5. Покрыть модуль простыми тестами или ручной проверкой через консольный скрипт.

---

## Этап 3. Логика моделей (`models.py`)

1. Описать dataclass-структуры: `Prompt`, `Model`, `Result`, `Setting`.
2. Реализовать получение списка активных моделей (`is_active = 1`).
3. Реализовать сохранение и выборку промтов (новый промт и выбор из истории).
4. Реализовать сохранение выбранных результатов в таблицу `results`.
5. Реализовать чтение/запись настроек программы.
6. Инкапсулировать работу с API-ключами: чтение имени переменной из БД, значение — из `.env`.

---

## Этап 4. Сетевой слой (`network.py`)

1. Реализовать базовый клиент для отправки промта в одну модель (httpx).
2. Добавить обработку ошибок: таймаут, HTTP-ошибки, отсутствие ключа в `.env`.
3. Реализовать параллельную или последовательную отправку промта во все активные модели.
4. Вернуть унифицированную структуру ответа: `model_name`, `response_text`, `error` (если есть).
5. Заложить расширяемость под разные типы API (OpenAI-совместимый формат как первый вариант).

---

## Этап 5. Каркас GUI (`main.py`)

1. Заменить минимальный прототип на полноценное главное окно.
2. Разбить интерфейс на зоны:
   - ввод/выбор промта;
   - кнопка «Отправить»;
   - таблица временных результатов;
   - кнопка «Сохранить».
3. Добавить меню или вкладки для управления моделями, историей промтов и настройками.
4. Подключить сигналы кнопок к обработчикам (пока с заглушками).

---

## Этап 6. Ввод и выбор промта

1. Добавить многострочное поле ввода промта.
2. Добавить выпадающий список или боковую панель с сохранёнными промтами из `prompts`.
3. При выборе сохранённого промта — подставлять текст и теги в форму.
4. При новом промте — сохранять запись в `prompts` (дата, текст, теги) перед или после отправки (по решению UX).

---

## Этап 7. Отправка запроса и временная таблица результатов

1. По нажатию «Отправить»:
   - очистить предыдущую временную таблицу в памяти;
   - получить активные модели из `models.py`;
   - вызвать `network.py` для каждой модели;
   - заполнить временную таблицу результатами.
2. Отобразить таблицу в GUI: колонки «Модель», «Ответ», «Выбрать» (чекбокс).
3. Показать индикатор загрузки на время запросов.
4. Отображать текст ошибки в ячейке ответа, если запрос не удался.

---

## Этап 8. Сохранение выбранных результатов

1. По нажатию «Сохранить»:
   - собрать строки с `selected = True`;
   - записать их в таблицу `results` через `db.py` / `models.py`;
   - очистить временную таблицу.
2. Показать пользователю сообщение об успешном сохранении или предупреждение, если ничего не выбрано.

---

## Этап 9. Новый запрос

1. При вводе нового промта или повторной отправке — полностью удалять временную таблицу.
2. После получения новых ответов — создавать таблицу заново.
3. Сбрасывать состояние чекбоксов.

---

## Этап 10. Управление моделями (GUI)

1. Экран/диалог со списком моделей из таблицы `models`.
2. Поля: имя, API URL, API ID, имя переменной окружения для ключа, активна/неактивна.
3. CRUD: добавление, редактирование, удаление, переключение `is_active`.
4. Не показывать и не хранить сами API-ключи — только имена переменных из `.env`.

---

## Этап 11. История промтов и сохранённых результатов

1. Экран просмотра таблицы `prompts` с поиском и сортировкой.
2. Экран просмотра таблицы `results` с привязкой к промту и модели.
3. Возможность повторно использовать промт из истории.

---

## Этап 12. Настройки программы

1. Экран настроек на основе таблицы `settings`.
2. Базовые параметры: таймаут запросов, путь к БД, тема интерфейса (опционально).
3. Сохранение изменений в SQLite.

---

## Этап 13. Дополнительные функции (опционально)

1. Поиск и сортировка во всех таблицах GUI.
2. Экспорт выбранных результатов в Markdown и JSON.
3. Поддержка разных типов моделей (OpenAI, DeepSeek, Groq и др.) через адаптеры в `network.py`.
4. Логирование запросов в файл (`logs/`).

---

## Этап 14. Сборка и финализация

1. Обновить `README.md`: установка, настройка `.env`, запуск.
2. Пересобрать исполняемый файл через PyInstaller (`ChatList.exe`).
3. Проверить работу exe без установленного Python.
4. Ручное тестирование полного сценария: ввод промта → отправка → выбор → сохранение → просмотр истории.

---

## Порядок зависимостей между этапами

```
Этап 1 → Этап 2 → Этап 3 → Этап 4
                ↘         ↙
                  Этап 5
                    ↓
         Этапы 6 → 7 → 8 → 9
                    ↓
         Этапы 10, 11, 12 (параллельно)
                    ↓
              Этапы 13, 14
```

Сначала — инфраструктура (БД, модули, сеть), затем — основной сценарий работы в GUI, после — вспомогательные экраны и опциональные возможности.
